You will receive one completed conversation episode as raw user/assistant messages.

In this prompt, an "episode" means one self-contained chunk of conversation made up only of the messages you are given.

Your task is to compress that episode into a structured JSON object.

Do not answer the user. Do not continue the conversation. Do not add commentary. Only return the JSON object described below.

Do not use markdown. Do not use code fences. Do not include any text before or after the JSON object.

### Input Schema
You will receive one JSON object with this shape:

{
  "messages": [
    {
      "role": "user" | "assistant",
      "content": string
    }
  ]
}

Interpretation:
- `messages` is the actual content you must compress.
- The messages are already provided in chronological order from earliest to latest.
- Treat the message order as meaningful. Do not assume the messages were shuffled or randomly sampled.

### Purpose
The output should preserve:
- a concise narrative summary of what happened
- a short one-sentence reference string
- a small set of topic keywords
- an importance score from 1 to 10
- the most useful exact details from the episode when concrete details are present

### Output Schema
Return exactly one JSON object with exactly these fields:

{
  "tier1_detail": string | null,
  "tier2_summary": string,
  "tier3_ref": string,
  "tier3_keywords": string[],
  "salience": integer
}

### Field Definitions

#### tier1_detail
- Either `null` or a string.
- Use this field for the most useful exact details that may need to be retrieved later.
- Prefer a non-null string when the episode includes concrete details worth preserving.
- Include exact commands, code, file names, error text, specific values, or precise step-by-step details when they appear and are relevant.
- Use `null` only when the episode genuinely contains no meaningful exact details to preserve.
- Never invent missing details.

#### tier2_summary
- Required.
- A concise narrative summary of the episode.
- Target 4-6 sentences maximum.
- Focus on:
  - what the topic was
  - what happened
  - what result, decision, fix, lesson, or unresolved question came out of it
- Use concrete language.
- Do not include unnecessary filler.

#### tier3_ref
- Required.
- A single-sentence bookmark for the episode.
- It should be short, specific, and suitable as a quick inline reference to what the episode was about.
- Do not include more than one sentence.

#### tier3_keywords
- Required.
- An array of exactly 5 short keywords or keyphrases.
- Use compact topic anchors such as tools, files, bugs, concepts, outcomes, people, or projects if they appear in the episode.
- Do not use full sentences.

#### salience
- Required.
- Integer from 1 to 10.
- This is the importance score for the episode.
- Use these guidelines:
  - 1-3: low-importance exchange, disposable small talk, or minor coordination
  - 4-6: normal useful discussion worth remembering in summarized form
  - 7-8: important progress, decisions, preferences, or notable outcomes
  - 9-10: highly important exact details, critical fixes, major decisions, precise implementation steps, or facts likely to be referenced later verbatim

### Compression Rules
- Preserve truth. Never hallucinate.
- Only use information present in the episode.
- If something is uncertain in the episode, preserve that uncertainty.
- If the episode did not reach a conclusion, state that clearly in the summary.
- If exact details are present but not central, leave them out of `tier2_summary`.
- Do not mention that you are summarizing, compressing, or producing JSON.

### Style Rules
- Be concise, specific, and literal.
- Prefer plain language.
- Do not use dramatic or decorative phrasing.
- Do not speak to the user directly.
- Do not use vague placeholders when the episode provides concrete details.

### Validation Checklist
Before finalizing your response, ensure:
- The response is valid JSON.
- All required fields are present.
- No extra fields are present.
- `tier3_keywords` has exactly 5 items.
- `salience` is an integer from 1 to 10.
- `tier3_ref` is exactly one sentence.
